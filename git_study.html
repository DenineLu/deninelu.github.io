Git与Github的基本学习

发布于 2019-10-03

1 Git 简介

Git是一个开源的分布式版本控制系统，可以有效、高速的处理从很小到非常大的项目版本管理。分布式相比于集中式的最大区别在于每个开发者都可以通过克隆，在本地机器上拷贝一个完整的Git仓库，并在本地仓库中修改，然后推送至远程库。它的特点有：版本切换便捷、独立的分支机制、完整性强等。

2 Git 安装

sudo apt-get install git
sudo apt-get install git-core

3 创建git仓库

1.cd至代码文件根目录下

2.初始化生成git本地仓库

git init

4 设置签名

1.设置用户名

git config (--global) user.name [username]

2.设置邮箱

git config (--global) user.email [useremail]

5 文件的提交

查看工作区、暂存区状态

git status

将工作区的目标文件添加至暂存区

git add [file_name]

将工作区的全部文件添加至暂存区

git add .

撤销git add操作

git rm --cache [file name]

撤销暂存区所有文件的git add操作

git rm -r --cache .

将暂存区的内容提交至本地库

git commit -m “description infomation” [file name]

将暂存区所有的文件都提交至本地库

git commit -m “description infomation” .

注：对于一个新文件的提交，必须经过git add操作，而对于文件的更新操作，可以直接git commit

查看历史记录

git log #标准输出
git log --pretty=oneline #以一行输出一条记录
git log --oneline #截取索引值的一部分（仍可唯一确定）
git reflog #截取索引值的一部分

6 版本的前进与后退

本质：存在HEAD指针指向当前版本，移动HEAD指针即可进行版本的前进与后退.

方法一：基于索引值（推荐）：可后退可前进

git reset --hard [局部索引值] #使用 git reflog 可得

方法二：使用^符号：只能后退

git reset --hard HEAD^^^ #几个^就后退几个版本

方法三：使用~符号：只能后退

git reset --hard HEAD~n #后退n个版本

reset的三个参数

--soft 参数：
仅在本地库移动HEAD指针.（即仅修改本地库）

--mixed 参数：
在本地库移动HEAD指针；
修改暂存区.

--hard 参数：
在本地库移动HEAD指针；
修改暂存区；
修改工作区.

删除文件并找回

在工作区删除了某文件后，可使用版本后退恢复文件。

git reset --hard [索引 / 指针]

比较文件差异

git diff [file_name] #该文件在工作区与暂存区的差异
git diff [HEAD] [file_name] #该文件在工作区与本地库某版本的差异

注：不指定文件名，则比较所有文件的差异

6 版本的前进与后退

分支：在版本控制的过程中，使用多条线同时推进多个任务，互不影响.

创建分支

git branch [branch_name]

查看分支

git branch -v

切换分支

git checkout [branch_name]

合并分支

将branch_one的内容合并到branch_two

git checkout branch_two
git merge branch_one

合并分支冲突

当两个分支的某个文件的某处内容不同，则合并时会发生冲突，需手工编辑文件.编辑好文件后，执行

git add [file_name]
git commit -m “description infomation”

注：此时git commit不能加文件名.

7 Github操作

在github上创建好仓库后，得到远程地址。

https://github.com/DenineLu/catkin_test.git

本地库推送至github

git push [remote address] [branch_name]

通常在本地将远程地址设置一个别名（一般为origin），那么命令可为

git push [origin] [branch_name]

克隆至本地

git clone [remote address]

效果：完整的把远程库下载到本地； 创建origin远程地址别名； 初始化本地库。

团队成员邀请

打开github项目仓库 -> Settings -> Collaborators -> 查找好友 -> 复制邀 请链接发送给好友

拉取远程库

pull = fetch + merge

方法一:

git fetch [origin] [remote branch_name]
git merge [origin/remote branch_name]

方法二：

git pull [origin] [remote branch_name]

PUSH冲突问题

A与B同时pull了远程库的最新版本，A和B都做了修改，A先push至远程库， B再push就会失败产生冲突。
注：如果不是基于远程库最新版所做的修改，不能推送，必须先拉取最新版。 拉取下来时如果进入冲突状态，则按照分支冲突解决操作即可。

跨团队协作

第三方进行Fork操作；
第三方pull至本地库进行开发，然后推送到远程库；
第三方发起Pull Requests；
原作者审核代码、合并。

SSH登录

删除home/.ssh 目录

rm -r ~/.ssh

生成.ssh秘钥目录

ssh-keygen -t rsa -C [github_account_email]

打开 .ssh/id_rsa.pub并复制内容

cat .ssh/id_rsa.pub

打开github账户Settings -> SSH and GPG keys -> new SSH key -> 粘贴内容 -> 复制远程库的SSH地址
在本地添加SSH地址的别名

git remote add [origin_ssh] [SSH_address]

测试Push

git push origin_ssh master
